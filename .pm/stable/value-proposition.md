# Value Proposition

## One-Line Value

Harness turns a normal coding agent into a governed autonomous product company: product definition, PM supervision, disciplined worker execution, git checkpoints, independent review, and resumable long-run state.

## For The First User

For solo builders using AI coding agents, Harness enables long-running project advancement while preserving control, reversibility, evidence, and maintainability.

## Why It Is Different

- It treats product definition as a contract, not chat context.
- It separates PM supervision from engineering execution.
- It requires worker reports, verification evidence, and acceptance review.
- It uses git history as part of the UX and safety model.
- It stops for product, risk, or evidence problems instead of silently drifting.

## Aha Moments

1. The user watches Harness act like a PM: clarify product, write tasks, spawn OpenCode, constrain the worker, review evidence, update state, and continue.
2. The user returns after being away and finds a highly complete product that fits or exceeds expectations, with an auditable trail of every iteration.

## Promise

You can leave the agent working without giving up control of the project.

## Anti-Promise

Harness does not promise blind fully autonomous shipping. It promises governed autonomy that knows when to stop.
