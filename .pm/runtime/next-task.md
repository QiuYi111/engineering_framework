# Worker Task Packet

## Objective

Implement the first deterministic PM runtime health-check slice for the dogfood `/goal` supervisor loop.

This task does not implement the full autonomous loop yet. It builds the first executable foundation the supervisor needs before safe long-running delegation: validate `.pm` runtime readiness, inspect loop state, verify report structure, and expose a CLI command the supervisor can use before and after worker iterations.

## Task

Add a small deterministic PM runtime module and CLI surface.

Required behavior:

1. Add a module such as `scripts/harness_runtime/pm_runtime.py`.
2. Validate required `.pm/stable` and `.pm/runtime` files exist and are non-empty.
3. Parse `.pm/runtime/state.yaml` and expose current stage, phase, loop iteration, readiness flags, worker mode, git branch policy, and next action.
4. Read `.pm/runtime/loop-control` and classify whether the supervisor should continue, stop, request user decision, or treat the state as invalid.
5. Validate `.pm/runtime/worker-report.md` structure when a real report exists. Placeholder reports should be reported as "not started", not accepted.
6. Inspect git branch and dirty files, but do not mutate git state.
7. Add a CLI command, for example `harness pm-status`, that prints a concise machine-readable-ish status summary and exits non-zero only for invalid/missing required runtime state.
8. Add unit tests using the Python standard library `unittest` if practical. Do not add new test dependencies.

This is a foundation task. Do not call OpenCode from the runtime yet. Do not implement full loop execution yet.
