## Task

Update `validate_worker_report()` so it validates required worker-report sections, not just the presence of markdown headings.

Required behavior:

1. Define the required worker-report sections based on the intern report template and acceptance rubric:
   - `Worker Report`
   - `Task summary` or `Objective`
   - `What was done` or `Changes`
   - `Changed files` or equivalent explicit changed-file section
   - `Commands run` or `Verification evidence`
   - `Test results` or explicit test evidence
   - `Acceptance criteria` or `Acceptance criteria checklist`
   - `Problems encountered`
   - `Deviations` or `Deviations from task`
   - `Evidence` or equivalent concrete evidence section
2. Return a structured result that distinguishes:
   - `not_started`
   - `placeholder`
   - `invalid`
   - `valid`
3. For `invalid`, include missing section names in the result.
4. Keep backwards-compatible CLI behavior if possible: `harness pm-status` should show invalid worker-report status and reason without making the whole PM runtime invalid. PM runtime validity should still be about required runtime state, not whether the latest worker report is acceptable.
5. Add or update tests for valid report, missing sections, placeholder report, and current real worker report shape if practical.

Do not implement full supervisor review yet. This task only strengthens deterministic report validation.
