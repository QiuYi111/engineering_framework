# Tasks Format Reference

The canonical format for `tasks.md` generated by `harness-tasks`.

## Header

```markdown
# Tasks: [Feature Name]

> Generated from spec.md + plan.md
> Risk Level: [leaf|branch|core|infra]

## Legend

- `[P]` — Parallel-safe (no dependencies on other tasks in same group)
- `[HITL]` — Requires human approval before proceeding
- `[AFK]` — Fully autonomous execution
- `→` — Depends on

## Risk Gates

| Level | Required Gates |
|-------|---------------|
| leaf  | lint + unit_test |
| branch | spec + plan + tests + review |
| core  | human_spec_review + architecture_review + rollback + security |
| infra | dry_run + human_approval + rollback + security |
```

## Task Entry Format

```markdown
### T001 [P] [AFK] [US1] Create order domain entity

**Files:**
- `src/ordering/domain/order.go` (create)
- `src/ordering/domain/order_test.go` (create)

**Acceptance:**
- Order struct with Create, AddItem, Submit methods
- Unit tests cover happy path and validation errors
- No external dependencies in domain layer

**Depends on:** (none)
```

## Section Structure

### Phase 1: Foundation
Tasks that create scaffolding, types, and interfaces.

### Phase 2: Vertical Slices
Each subsection = one user story, sliced vertically.

```markdown
#### Slice A: [US1] Create Order

T001 [P] [AFK] [US1] Write test for order creation
T002 [P] [AFK] [US1] Implement order creation
T003 [AFK] [US1] Wire order creation to HTTP endpoint
```

### Phase 3: Integration
Cross-slice wiring, shared infrastructure.

### Phase 4: Verification
End-to-end tests, harness compliance, `make verify`.

## Dependencies Graph

```markdown
## Dependencies

T001 → (none)
T002 → T001
T003 → T002
T004 [P] → (none)
T005 [P] → (none)
T006 → T003, T004, T005
```

## Parallel Execution Plan

```markdown
## Parallel Execution

| Wave | Tasks | Notes |
|------|-------|-------|
| 1    | T001, T004, T005 | Independent slices |
| 2    | T002 | Waits for T001 |
| 3    | T003 | Waits for T002 |
| 4    | T006 | Integration — waits for all |
```

## Checkpoints

```markdown
## Checkpoints

### Checkpoint 1: Domain Layer Complete
- After: T002
- Verify: `make verify-ai`
- Gate: Domain tests green, no lint errors
- Continue: T003–T005

### Checkpoint 2: Integration Ready
- After: T006
- Verify: `make verify && make verify-ai`
- Gate: All acceptance criteria met
- Continue: T007 (verification)
```

## Rules

1. Every file path is exact — no wildcards, no "etc."
2. Create/modify verbs are explicit: `(create)`, `(modify)`, `(delete)`
3. Acceptance criteria reference spec requirements by ID
4. Risk classification uses leaf/branch/core/infra terminology
5. HITL gates match the risk level required gates
